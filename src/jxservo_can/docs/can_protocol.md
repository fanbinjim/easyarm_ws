# 巨蟹智能关节模组 CANFD/CANopen 协议摘要

资料来源：

- PR0002 关节模组 CANFD/CANopen 使用协议简易说明书，Rev. 1.0，2026-05-11。
- TN0001 MIT 模式 CANopen 使用教程，Rev. 1.0，2026-05-11。
- FC0001 V2.0报错代码&SDO读取，Rev. 1.0，2026-05-11。

## 信息完整性判断

三份文档的信息足够实现 `jxservo_can` 的基础通信驱动，包括 SocketCAN/CANFD 参数、SDO 读写、DS402 使能流程、自定义快控控制帧、自定义反馈帧、MIT 控制入口、MIT payload 组包、错误码读取和主要单位换算。

仍需实机确认的点：

- PR0002 的 CANopen PDO 配置章节来自 Word/OCR 整理，文档自身说明存在错位风险，建议通过实机回读对象字典确认。
- SDO 多字节字段是小端；自定义快控和自定义反馈示例表现为大端。实现时必须严格区分，并用实机验证。
- `606C` 速度 SDO 示例显示 `0x01F9 -> 50.5 RPM`，暗示比例为 `0.1 RPM/count`，但文档未显式写出。
- `6064` SDO 实际位置示例只演示低 16 bit，完整 32 bit 有符号、多圈行为和过零策略需要实机确认。
- NMT 示例中 `81 01` 被标注为“开启节点”，但标准 CANopen 中 `0x81` 是 reset node；启动节点应使用 `01 <node_id>` 或广播 `01 00`。
- MIT 的 `P_MAX/V_MAX/T_MAX` 来自设备当前配置，驱动需要由调用者或配置文件提供，不能从这三份文档唯一确定。

## 总线与设备

- 支持 CAN 2.0 和 CAN FD，协议基于 CANopen 402/CiA 402/DS402。
- 默认 CAN FD 配置：标准帧，仲裁段 `1 Mbps`，数据段 `5 Mbps`，BRS 数据段加速开启。
- 执行器 ID 范围：`0x01..0x7F`。
- 上电流程：Boot 约 `300 ms`，App 初始化约 `2 s`。
- 初始化完成后设备发送 `0x700 + Dev_ID`，数据 `00`。
- 看门狗数据帧间隔应小于 `500 ms`，否则电机会自锁。
- 上伺服后应延时 `100 ms` 再发送新位置运动命令。
- 抱闸打开和关闭均约有 `100 ms` 延时。

## DS402 控制流程

以 `Dev_ID = 1` 为例，SDO CAN ID 为 `0x601`。

| 步骤 | SDO 数据 | 说明 |
| --- | --- | --- |
| 准备启动 | `2B 40 60 00 06 00 00 00` | `6040h = 0x0006` |
| 松开抱闸 | `2B 40 60 00 07 00 00 00` | `6040h = 0x0007` |
| 电机使能 | `2B 40 60 00 0F 00 00 00` | `6040h = 0x000F` |
| 清错 | `2B 40 60 00 80 00 00 00` | `6040h = 0x0080` |

状态字示例：

- `0x0021`：Ready to switch on。
- `0x0023`：Switch on，抱闸释放。
- `0x0027`：Operation enabled。
- `0x0008`：Fault。

模式值：

| 模式 | `6060h` 值 | 说明 |
| --- | --- | --- |
| PP | `1` | Profile Position |
| PV | `3` | Profile Velocity |
| CSP | `7` | Cyclic Synchronous Position，PDO 配置示例使用 |
| 电流模式 | `10` | 文档 SDO 电流模式示例使用 |
| MIT | `-64` / `0xC0` | 厂家自定义 MIT 模式 |

## 自定义单轴快控

主站发送给执行器：标准 CAN FD 帧，`StdID = 0x100 + Dev_ID`，`DLC = 7`。

Byte0 控制位：

| 位 | 含义 |
| --- | --- |
| Bit7 | 使能控制，`0` 下使能，`1` 上使能 |
| Bit6 | 抱闸控制，`0` 抱闸吸合，`1` 抱闸释放 |
| Bit5 | 清除错误，`0` 默认，`1` 复位错误 |
| Bit4..1 | 控制模式：`1` PP，`2` PV，`3` CSP，`4` CSV，`5` 电流环 |
| Bit0 | 预留 |

Byte1..6 参数为 3 个 `int16`，文档示例使用大端：

| 字节 | 含义 |
| --- | --- |
| Byte1..2 | 目标参数 1。位置模式为输出端目标位置，`-32768..32767` 对应 `-180..180 deg`；速度模式为电机端目标速度，单位 `RPM`；电流模式为目标 Iq 电流，单位 `mA` |
| Byte3..4 | 目标参数 2。PP/PV 模式为轮廓加减速，范围 `0..20000 RPM/s` |
| Byte5..6 | 目标前馈参数。位置模式为输出端轮廓速度，其余模式预留 |

示例，`Dev_ID = 1`：

| CAN ID | Data | 说明 |
| --- | --- | --- |
| `0x101` | `C2 00 00 07 D0 00 05` | PP，到 `0 cnt`，加减速 `2000`，速度 `5` |
| `0x101` | `C2 40 00 03 E8 00 08` | PP，到 `16384 cnt`，加减速 `1000`，速度 `8` |
| `0x101` | `C2 C0 00 03 E8 00 08` | PP，到 `-16384 cnt`，加减速 `1000`，速度 `8` |
| `0x101` | `C4 01 F4 03 E8 00 00` | PV，速度 `500 RPM`，加减速 `1000` |
| `0x101` | `C4 FE 0C 03 E8 00 00` | PV，速度 `-500 RPM`，加减速 `1000` |
| `0x101` | `CA 03 E8 00 00 00 00` | 电流模式，电流 `1000 mA` |
| `0x101` | `CA FC 18 00 00 00 00` | 电流模式，电流 `-1000 mA` |
| `0x101` | `C6 00 32 00 00 00 00` | CSP，到 `50 cnt` |
| `0x101` | `C6 00 64 00 00 00 00` | CSP，到 `100 cnt` |

## 自定义反馈帧

执行器发送给上位机：标准 CAN FD 帧，`StdID = 0x300 + Dev_ID`，`DLC = 12`。

触发方式：

- 上位机发送同步帧 `0x80`。
- 执行器收到多轴控制报文。
- 执行器收到单轴控制报文。

字段格式，文档示例使用大端：

| 字节 | 类型 | 说明 |
| --- | --- | --- |
| Byte0..1 | `int16` | 负载端实际位置，`-32768..32767` 对应 `-180..180 deg` |
| Byte2..3 | `int16` | 电机端实际速度，单位 `RPM` |
| Byte4..5 | `int16` | 实际 Iq 电流，单位 `mA` |
| Byte6..7 | `uint16` | 错误代码 |
| Byte8..9 | `int16` | 电机线圈温度，单位 `0.1 degC` |
| Byte10 | `uint8` | 控制模式反馈，`1` PP，`2` PV，`3` CSP，`4` CSV，`5` 电流环，`6` MIT |
| Byte11 Bit7 | `bool` | 使能状态，`0` 下使能，`1` 上使能 |
| Byte11 Bit6 | `bool` | 抱闸状态，`0` 抱闸吸合，`1` 抱闸释放 |
| Byte11 Bit5 | `bool` | 报错状态，`0` 正常，`1` 报错 |
| Byte11 Bit4 | `bool` | 位置到位，`0` 运行中，`1` 到位 |
| Byte11 Bit3..0 | - | 预留 |

示例：

```text
Rx 0x301 DLC 12: 09 E1 FD E5 FF 51 00 00 00 F0 01 C0
```

解析：

- `09 E1`：约 `13 deg`。
- `FD E5`：`-539 RPM`。
- `FF 51`：`-175 mA`。
- `00 00`：无报错。
- `00 F0`：`24.0 degC`。
- `01`：PP 模式。
- `C0`：上使能、抱闸释放、正常、运行中。

## 单位换算

自定义快控和反馈：

- 负载端位置原始值 `int16`，`-32768..32767` 对应 `-180..180 deg`。
- `rad = raw / 32768.0 * pi`。
- `raw = round(rad / pi * 32768.0)`，发送前应限幅到 `[-32768, 32767]`。
- 电机端速度单位 `RPM`，`rad/s = rpm * 2*pi/60`。若需要负载端速度，需除以减速比。
- 电流单位 `mA`。
- 温度单位 `0.1 degC`。

SDO：

- `6078` 实际电流单位 `mA`。
- `2662/2663` 温度单位 `0.1 degC`。
- `6064` 实际位置文档示例：一圈 `65536 cnt`，约 `182 cnt/deg`，对应 `-180..180 deg`。
- `606C` 实际速度示例暗示 `0.1 RPM/count`，需实机确认。

编码器：

- 电机端 `14 bit = 16384 cnt/rev`。
- 负载端 `16 bit = 65536 cnt/rev`。
- 过零会发生 `65535 -> 0` 或 `0 -> 65535` 跳变，全闭环累计位置需要做过零补偿。

## CANopen PDO 配置摘录

PR0002 明确提示 PDO 部分不是完整预置映射，且原始 Word/OCR 代码片段存在错位风险。本节只作为实现参考。

典型流程：

1. NMT/心跳配置。
2. 禁用 TPDO/RPDO。
3. 配置通信参数和映射对象。
4. 重新启用 PDO。
5. 设置模式。
6. 启动节点并周期发送 SYNC。

示例片段：

- 上电心跳：`0x701 DLC 1 Data 00`。
- 配置心跳时间：`0x601 Data 2B 17 10 00 C8 00 00 00`，`1017h = 200 ms`。
- 禁用 TPDO1：写 `1800:01 = 0x80000181`。
- 设置 TPDO1 传输类型：写 `1800:02 = 1`。
- 清空 TPDO1 映射：写 `1A00:00 = 0`。
- 映射 TPDO1：`6041:00/16bit`、`6078:00/16bit`、`6064:00/32bit`。
- 启用 TPDO1：写 `1800:01 = 0x00000181`，TPDO1 CAN ID 为 `0x181`。
- 禁用 RPDO1：写 `1400:01 = 0x80000201`。
- 设置 RPDO1 传输类型：写 `1400:02 = 1`。
- 清空 RPDO1 映射：写 `1600:00 = 0`。
- 映射 RPDO1：`6040:00/16bit`、`6071:00/16bit`、`607A:00/32bit`。
- 启用 RPDO1：写 `1400:01 = 0x00000201`，RPDO1 CAN ID 为 `0x201`。

## 安全注意事项

- 首次调试不要关闭看门狗和限位保护。
- 关闭看门狗/限位保护的 SDO `23 50 26 00 01 00 00 00` 只能用于明确风险的调试场景。
- 设置正负限位时电机应处于失能状态，设置后需保存到 Flash。
- 零位标定时电机需要处于失能状态，并按文档要求完成标定复位和重新使能流程。
- 发送使能、抱闸释放、运动、MIT 力矩类命令前，必须确认机械限位、急停、电源限流和负载状态安全。
